# Plan Format Guide
Standard Markdown structure for implementation plans.

## Structure
- **Title** (H1)
- **Status Summary** (Bullet points)
- **Tasks** (Task list with checkboxes)

## Task Formatting
- `[ ] Task description` - Pending
- `[x] Task description` - Completed
- `[-] Task description` - Blocked
- `[~] Task description` - In-progress

## Examples
```markdown
# My Feature Plan

- Total Tasks: 3
- Completed: 1

## Core Implementation
- [x] Create API endpoint
- [~] Add validation logic
- [ ] Write unit tests
```
