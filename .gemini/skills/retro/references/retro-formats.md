# Retrospective Formats Reference

Templates, git commands, and pattern analysis for sprint retrospectives.

## Git Metrics Commands

```bash
# Total commits in period
git log --oneline --since="2024-01-08" --until="2024-01-14" | wc -l

# Commits by type (conventional commits)
git log --format="%s" --since="7 days ago" | grep -c "^feat:"
git log --format="%s" --since="7 days ago" | grep -c "^fix:"
git log --format="%s" --since="7 days ago" | grep -c "^refactor:"
git log --format="%s" --since="7 days ago" | grep -c "^chore:"

# Contributor breakdown
git shortlog -sn --since="7 days ago"

# Files and line changes
git log --since="7 days ago" --format="" --shortstat | \
  awk '/files changed/{f+=$1; a+=$4; d+=$6} END{print f" files, +"a" -"d}'

# Commit list with date
git log --format="%ad %s" --date=short --since="7 days ago"

# PRs merged (GitHub CLI)
gh pr list --state merged --base main --search "merged:>2024-01-08"
```

## Standard Retrospective Format (4Ls)

```markdown
# Sprint Retrospective — {period}

**Period:** {from} to {to}
**Team:** {members or "solo"}
**Generated:** {date}

---

## Metrics

| Metric | Value |
|--------|-------|
| Total Commits | {n} |
| Features (feat:) | {n} ({pct}%) |
| Bug Fixes (fix:) | {n} ({pct}%) |
| Refactors | {n} ({pct}%) |
| Files Changed | {n} |
| Lines Added/Removed | +{n} / -{n} |

**Commit Type Distribution:**
```
feat:     ████████░░ 40%
fix:      ██████░░░░ 30%
refactor: ████░░░░░░ 20%
chore:    ██░░░░░░░░ 10%
```

---

## Went Well
- **{item}** — {evidence from git/journal}
- **{item}** — {metric supporting this}

## What to Improve
- **{problem}** — Impact: {what it affected}
- **{problem}** — Impact: {velocity lost, bugs introduced, etc.}

## Action Items (max 3)
| # | Action | Owner | Due |
|---|--------|-------|-----|
| 1 | {SMART action} | {name} | {sprint N+1} |
| 2 | {SMART action} | {name} | {date} |
| 3 | {SMART action} | {name} | {date} |

## Patterns Detected
- {pattern with evidence}

---
*Generated by gk-retro*
```

## Velocity Health Thresholds

| Metric | Healthy | Warning | Critical |
|--------|---------|---------|---------|
| fix: ratio | < 25% | 25-40% | > 40% |
| feat: ratio | > 40% | 20-40% | < 20% |
| Commits/day | 3-8 | 1-2 or 9-15 | 0 or >15 |
| Single contributor % | < 70% | 70-85% | > 85% |

## Pattern Library

### High Fix Ratio (>40%)
- **Meaning:** Quality debt or premature shipping
- **Evidence:** Count of fix: commits / total commits
- **Action:** Add unit test requirement for PRs; root cause top 3 recurring bugs

### Low Feature Velocity
- **Meaning:** Blocked, in meetings, or heavy tech debt
- **Evidence:** feat: < 20% with normal commit count
- **Action:** Review blockers log; reduce WIP limit

### Commit Clustering (all commits on 2-3 days)
- **Meaning:** Work not flowing — blocked or batching
- **Evidence:** `git log --format="%ad" --date=short` shows clusters
- **Action:** Daily standups to surface blockers earlier

### Solo Contributor
- **Meaning:** Bus factor risk
- **Evidence:** One author > 85% of commits
- **Action:** Pair programming, explicit knowledge transfer sessions

### Churn Indicator
- **Meaning:** High lines removed vs added = heavy refactor or instability
- **Evidence:** lines removed > lines added by 2x
- **Action:** Evaluate if refactor was planned or reactive

## Mode Configurations

### --week Mode
```
since: "7 days ago" (or last Monday 00:00)
until: "now" (or last Sunday 23:59)
label: "Week of {Mon date}"
```

### --sprint Mode
```
since: "14 days ago" (or sprint start date from config)
until: "now" (or sprint end date)
label: "Sprint {N} — {dates}"
```

### --custom Mode
```
since: user-provided ISO date or relative string
until: user-provided ISO date or "now"
label: "Custom: {since} to {until}"
```

## Action Item SMART Templates

| Pattern | Action Template |
|---------|----------------|
| High fix ratio | "Add mandatory unit tests for {module} PRs — enforced via CI check by {date}" |
| Missing tests | "Reach {N}% test coverage for {module} using {test framework} by sprint end" |
| Slow reviews | "Cap PR review SLA at 24h — add GitHub auto-assign by {date}" |
| Single contributor | "Pair {person} with {person2} on {component} — 2 sessions this sprint" |
| Documentation gaps | "Document {API/module} in {docs/file} before next sprint planning" |

## Week vs Sprint Report Differences

| Section | Week Report | Sprint Report |
|---------|-------------|---------------|
| Metrics scope | 7 days | 14 days |
| Action items | 1-2 max | 3 max |
| Velocity benchmark | vs last week | vs sprint goal |
| Patterns | Daily flow | Sprint-level trends |
| Format | Brief (< 1 page) | Full retrospective |
